# Roadmap

## Overview

This roadmap outlines plausible next steps for Alfa Hawk as an open-source forensic workstation and hosted evidence-analysis platform.

## Near-Term Priorities

- complete the open-source core vs platform boundary cleanup
- rationalize legacy AI configuration paths
- improve schema validation and typed contracts across the pipeline
- expand automated test coverage for API and report generation
- harden production deployment guidance

## Product and UX

- richer timeline filtering and comparative event review
- multi-frame evidence boards
- stronger chain-of-custody support
- collaborative investigation workspace features
- case-level annotations and reviewer notes

## AI and Analysis

- multi-language support
- advanced object detection model integration
- deeper audio and transcript analysis workflows
- confidence calibration improvements
- explainable evidence-link generation

## Platform and Enterprise

- on-premise enterprise deployment profiles
- SSO and RBAC support
- audit logging and evidence access controls
- externalized Redis-backed session and rate-limit storage
- queue-based scaling for longer-running analyses

## Reporting

- more configurable PDF templates
- export variants for internal review vs legal handoff
- stronger report integrity verification workflows

## Developer Ecosystem

- formal OpenAPI specification
- SDKs or client helpers
- plugin-style provider support
- core package extraction into a reusable installable module

## Strategic Direction

The long-term direction is to preserve Alfa Hawk as:

- a developer-friendly open-source forensic core
- a professionally operated hosted platform
- an enterprise-ready workstation for evidence review and structured reporting
